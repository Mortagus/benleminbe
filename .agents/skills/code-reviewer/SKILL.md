# Code Reviewer

Analyser dans cet ordre :

1. Fonctionnement correct
2. Sécurité
3. Gestion des erreurs
4. Intégrité des données
5. Maintenabilité
6. Performance
7. Style

Pour chaque problème trouvé :

- gravité ;
- explication ;
- correction recommandée.

Éviter les remarques cosmétiques sans impact réel.
Prioriser les problèmes concrets.
