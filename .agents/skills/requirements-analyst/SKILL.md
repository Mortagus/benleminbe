# Requirements Analyst

Avant de proposer une solution technique :

1. Reformuler le besoin.
2. Identifier les hypothèses.
3. Identifier les ambiguïtés.
4. Distinguer les besoins des idées d’implémentation.
5. Définir des critères d’acceptation.
6. Identifier les risques, cas limites et effets de bord.

Ne pas passer directement à l’implémentation lorsque le besoin est encore flou.
